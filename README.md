**概要**

このリポジトリは、Eoi氏の解説ページ（ノート）を基にしつつ、Glitchサービス終了に伴いローカル環境で実行できるよう一部改変を加えたDiscord用スレッド自動作成ボットです。

**謝辞**

本プロジェクトの実装は、Eoi氏による解説ページ（ https://note.com/exteoi/n/n00342a623c93 ）を参照して行っています。解説および参考資料としてEoi氏のNoteに深く感謝します。

**変更点（Glitch終了に伴う対応）**

- Glitch上で動作していた元実装をローカル実行へ移行するため、一部の環境設定と起動フローを変更しました。
- 外部サービス依存やファイル配置をローカル向けに調整しています。

**スレッド自動作成機能**

- `ThreadCreate.mjs` はEoi氏のサンプルには含まれておらず、本リポジトリでオリジナルにて作成した主要機能ファイルです。
- 主な機能:
  - 特定のメッセージに反応してスレッドを作成する。
  - 募集に参加した人数が所定数に達したら募集を終了する（スレッドのクローズ）。
- 上記に合わせて `main`（起動ロジック）にも一部改変を加えています。

**募集用スレッドの詳しい挙動（オンラインゲームコミュニティ向け）**

- 概要: サーバー内での募集メッセージ（例: パーティ募集）に対して自動でスレッドを作成し、参加者を集めるためのフローを提供します。

- トリガー: 投稿本文に `@` に続けて数字が含まれると、BOT が元メッセージに対してスレッドを作成します。慣習例: `ゲーム募集@3`（3人募集）

- スレッド作成時の振る舞い:
  - BOT は元メッセージを参照して新しいスレッドを作成し、スレッド内で募集を開始します。
  - スレッド作成時に募集主（元メッセージ投稿者）は募集人数を選択できます（初期値は本文中の数字を参照）。

- 参加方法:
  - 参加希望者は該当スレッドに対してメッセージを送ることで参加表明します。

- 募集の自動チェックと終了:
  - BOT はスレッド内のメンバー数を監視します。
  - 募集が成立する条件は「スレッドのメンバー数 == 募集人数 + 2（BOT と募集主）」です。
  - 条件を満たした際、BOT は募集主へ通知を送ります（スレッド内メンション）。
  - 募集を〆る際に、BOT が元メッセージへ指定のリアクションを付けるオプションを選択できます（❌を付与して募集完了を可視化）。

- 例フロー: `ゲーム募集@3` と投稿した場合
  1. BOT が元メッセージから `@+3` を認識してスレッドを作成。
  2. スレッド内で募集主が募集人数を確認・選択（必要であれば変更）。
  3. 参加者がスレッドへメッセージで参加。
  4. BOT がメンバー数を監視し、参加者数が `3 + 2`（募集主＋BOT）に達すると募集主へ通知。
  5. 必要に応じて BOT が元メッセージへリアクションを付け、募集を閉じる。

**ローカル環境での実行手順**

前提: Node.js がインストールされていること（推奨: Node 16+）。Windows 環境での手順例を示します。

1. 依存関係をインストール:

```powershell
npm install
```

2. 環境変数を設定:

- Discord Bot トークンなどを `.env` ファイルで用意してください。例:

```text
DISCORD_TOKEN=your_token_here
CLIENT_ID=your_client_id
GUILD_ID=your_guild_id
```

3. 起動:

```powershell
node main.mjs
```

4. 必要に応じてコマンド登録:

```powershell
node regist-commands.mjs
```

**注意点**
- 環境差分（OSやNodeバージョン）により動作が変わる場合があります。問題が出たら Node のバージョンを揃えて再確認してください。
